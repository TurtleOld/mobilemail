# Настройка GitHub Pages для документации

## Автоматическая публикация

Документация автоматически публикуется на GitHub Pages при пуше в ветку `main` через GitHub Actions.

## Первоначальная настройка

1. Перейдите в **Settings** → **Pages** вашего репозитория на GitHub
2. В разделе **"Source"** выберите **"GitHub Actions"**
3. Сохраните настройки

## URL документации

После настройки документация будет доступна по адресу:
**https://turtleold.github.io/mobilemail/**

## Как это работает

1. При пуше в ветку `main` запускается workflow `.github/workflows/docs.yml`
2. Workflow генерирует документацию через `./gradlew dokkaHtml`
3. Документация загружается как artifact
4. GitHub Pages автоматически публикует artifact

## Ручной запуск

Вы можете вручную запустить публикацию документации:

1. Перейдите в **Actions** → **Deploy Documentation**
2. Нажмите **"Run workflow"**
3. Выберите ветку `main`
4. Нажмите **"Run workflow"**

## Проверка статуса

1. Перейдите в **Actions** вкладку репозитория
2. Найдите workflow **"Deploy Documentation"**
3. Проверьте статус выполнения

## Устранение проблем

### Документация не публикуется

- Убедитесь, что в Settings → Pages выбран источник "GitHub Actions"
- Проверьте, что workflow выполнился успешно в Actions
- Убедитесь, что у репозитория есть права на публикацию Pages

### Документация не обновляется

- Проверьте, что изменения были запушены в ветку `main`
- Убедитесь, что изменены файлы в `app/src/**` или `app/build.gradle.kts`
- Проверьте логи workflow в Actions
